Personalization memory
for every AI app.

personiq extracts what matters about each user, stores it locally, and silently prepends it to every future conversation - one adapter, any LLM, two lines of code.

Get started → View on GitHub

How it works

A four-stage pipeline runs transparently on every conversation, alongside your existing LLM setup.

01
Extract
Your LLM reads the conversation and identifies durable facts skills, goals, preferences, style.
02
Store
Facts persist in a local SQLite database with vector embeddings and BM25 indexes.
03
Retrieve
Hybrid search fuses cosine similarity, BM25 keyword scoring, and recency decay via RRF.
04
Inject
Retrieved context is prepended to your system prompt before each LLM call.

Installation

Install the base package together with your LLM provider in one step.

bash
# pick one provider
pip install personiq[groq]        # Groq llama, gemma, mixtral
pip install personiq[openai]      # OpenAI
pip install personiq[anthropic]   # Anthropic Claude
pip install personiq[ollama]      # local, no API key
pip install personiq[mistral]     # Mistral
pip install personiq[all]         # Groq + OpenAI + Anthropic
💡
No external database personiq stores everything in a local SQLite file. No Redis, no Postgres, no cloud dependencies beyond your LLM provider.

Quickstart

  1. Install

    bash
    pip install personiq[groq]

  2. Create the adapter

    python
    from langchain_groq import ChatGroq
from personiq import PersoniqAdapter

piq = PersoniqAdapter(llm=ChatGroq(model="llama-3.1-8b-instant"))

  3. Add to your conversation loop

    python
    from langchain_core.messages import HumanMessage, SystemMessage

async def chat(user_id: str, message: str):
    # retrieve what we know about this user
    ctx = piq.persona(user_id, message)
    system = f"{ctx}\nYou are a helpful assistant." if ctx else "You are a helpful assistant."

    msgs = [SystemMessage(content=system), HumanMessage(content=message)]
    response = await piq._llm.ainvoke(msgs)

    # extract and store new memories (fire-and-forget)
    await piq.learn(user_id, msgs + [response])
    return response.content

After the first conversation, every subsequent call will include what personiq learned about that user.

Integration styles

Four patterns from the same adapter. Choose based on your architecture.

✦
Recommended for LangGraph Two nodes, zero boilerplate.

builder.add_node("personiq_save", piq.save_node)  # after chat
builder.add_edge("personiq_load", "chat")
builder.add_edge("chat", "personiq_save")

async def chat(state: PersoniqState):
    ctx = state.get("memory_context", "")

    msgs = [SystemMessage(content=system)] + list(state["messages"])
    resp = await llm.ainvoke(msgs)
    return {"messages": [resp]}

Context modes

Two output formats. Set once on the adapter or override per call.

Structured bullet list. Best for dashboards, technical assistants, recommendation engines.

output
[personiq: what I know about this user]

Technical background:
  • User is an experienced Go developer
  • User works with PostgreSQL and Redis

Goals:
  • User is building a distributed payment service

[end of personiq context]

Natural-language paragraph. Best for conversational chatbots - the assistant responds naturally without reciting a database.

output
You already know this user. Here is what you know about them:
They work with Go and PostgreSQL. They're currently focused on building
a distributed payment service. They prefer concise, direct answers.
Use this naturally don't recite it back, just let it shape your response.
python
# set globally
piq = PersoniqAdapter(llm=llm, mode="persona")

# override per call - these always use their own format
ctx = piq.context(user_id, query)   # always bullets
ctx = piq.persona(user_id, query)   # always natural language

LangGraph

Two nodes wired before and after your chat node. PersoniqState extends MessagesState and adds user_id and memory_context fields.

python
from langchain_groq import ChatGroq
from langchain_core.messages import HumanMessage, SystemMessage
from langgraph.graph import END, StateGraph
from personiq import PersoniqAdapter, PersoniqState

llm = ChatGroq(model="llama-3.1-8b-instant")
piq = PersoniqAdapter(llm=llm, mode="persona")

async def chat(state: PersoniqState) -> dict:
    ctx = state.get("memory_context", "")
    system = f"{ctx}\nYou are a helpful assistant." if ctx else "You are a helpful assistant."
    msgs = [SystemMessage(content=system)] + list(state["messages"])
    resp = await llm.ainvoke(msgs)
    return {"messages": [resp]}

builder = StateGraph(PersoniqState)
builder.add_node("personiq_load", piq.load_node)
builder.add_node("chat", chat)
builder.add_node("personiq_save", piq.save_node)
builder.set_entry_point("personiq_load")
builder.add_edge("personiq_load", "chat")
builder.add_edge("chat", "personiq_save")
builder.add_edge("personiq_save", END)
graph = builder.compile()

result = await graph.ainvoke({
    "user_id": "alice",
    "messages": [HumanMessage(content="I'm a backend engineer using Go.")],
})

LangChain / LCEL

python
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from personiq import PersoniqAdapter

llm = ChatOpenAI(model="gpt-4o-mini")
piq = PersoniqAdapter(llm=llm)
chain = ChatPromptTemplate.from_messages([
    ("system", "{system}"),
    ("human", "{input}"),
]) | llm | StrOutputParser()

async def ask(user_id: str, question: str) -> str:
    ctx = piq.persona(user_id, question)
    system = f"{ctx}\nYou are a helpful assistant." if ctx else "You are a helpful assistant."
    answer = await chain.ainvoke({"system": system, "input": question})
    await piq.learn(user_id, [HumanMessage(content=question)])
    return answer

FastAPI

python
from fastapi import FastAPI
from pydantic import BaseModel
from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage, SystemMessage
from personiq import PersoniqAdapter

app = FastAPI()
piq = PersoniqAdapter(llm=ChatOpenAI(model="gpt-4o-mini"), mode="persona")

class ChatReq(BaseModel):
    user_id: str
    message: str

@app.post("/chat")
async def chat(req: ChatReq):
    ctx = await piq.apersona(req.user_id, req.message)
    system = f"{ctx}\nYou are a helpful assistant." if ctx else "You are a helpful assistant."
    msgs = [SystemMessage(content=system), HumanMessage(content=req.message)]
    resp = await piq._llm.ainvoke(msgs)
    await piq.learn(req.user_id, msgs + [resp])
    return {"reply": resp.content}

@app.delete("/users/{user_id}/memories")
async def forget(user_id: str):
    return {"deleted": piq.forget(user_id)}

OpenAI SDK (direct)

Use inject() to work with the raw OpenAI message format without constructing a system prompt manually.

python
import openai
from langchain_openai import ChatOpenAI
from personiq import PersoniqAdapter

client = openai.AsyncOpenAI()
piq = PersoniqAdapter(llm=ChatOpenAI(model="gpt-4o-mini"))

async def chat(user_id: str, text: str):
    messages = [
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": text},
    ]
    # inject() finds the system message and prepends context
    messages = piq.inject(messages, user_id=user_id, query=text)
    resp = await client.chat.completions.create(model="gpt-4o-mini", messages=messages)
    return resp.choices[0].message.content

PersoniqAdapter

The primary class. Instantiate once per application; share across all users.

PersoniqAdapter(llm, config=None, mode="context", user_id_key="user_id")
Personalization memory adapter. One instance per application, shared across all users.
llm
Any LangChain BaseChatModel. Used for memory extraction and optional query expansion.
config
Optional PersoniqConfig. Sensible defaults applied if omitted.
mode
"context" for structured bullets, "persona" for natural language. Can be overridden per call.
user_id_key
LangGraph state key holding the user ID. Default: "user_id".

Context retrieval

piq.context(user_id: str, query: str) → str
Retrieve personalization context as a structured bullet list. Returns "" when no memories exist, safe to prepend unconditionally. Synchronous.
piq.persona(user_id: str, query: str) → str
Retrieve context as a natural-language persona paragraph. Better for conversational chatbots. Synchronous.
await piq.acontext(user_id, query) → str # async variant of context()
await piq.apersona(user_id, query) → str # async variant of persona()
Async variants. Prefer these in async applications.
piq.inject(messages: list[dict], user_id: str, query: str, mode=None) → list[dict]
Inject context into an OpenAI-style messages list. Finds the first system message and prepends context; inserts a new system message at index 0 if none exists.

Memory extraction

await piq.learn(user_id: str, messages: list[BaseMessage]) → int
Extract and store new memories from a conversation turn. Call after each assistant response. Pass the full conversation history for best results. Returns number of new memories (0 when async_extraction=True).
piq.learn_sync(user_id: str, messages: list[BaseMessage]) → int
Synchronous version of learn(). Blocks until extraction completes. Use only when async is unavailable.

LangGraph nodes

piq.load_node # async callable
Reads state[user_id_key] and state["messages"]. Writes retrieved context to state["memory_context"]. Attach before your chat node.
piq.save_node # async callable
Extracts and saves new memories fire-and-forget. Never blocks the response path. Attach after your chat node.

Memory management

piq.memories(user_id: str, category=None) → list[Memory]
List stored memories sorted by importance. Optional category filter: "preference", "goal", "technical", "context", "style", "personal".
piq.forget(user_id: str) → int
Delete all memories for a user. Returns number of deleted records. Use for GDPR compliance or account deletion.
piq.count(user_id: str) → int
Return total number of stored memories for a user.

PersoniqConfig

All settings in one dataclass. Every field can also be set via a PERSONIQ_* environment variable.

python
from personiq import PersoniqConfig

config = PersoniqConfig(
    db_path="./myapp.db",            # SQLite file path
    embedding_backend="local",       # "local" | "openai"
    embedding_model="all-MiniLM-L6-v2",
    top_k=5,                          # memories returned per query
    similarity_threshold=0.20,        # minimum score to include
    semantic_weight=0.60,             # hybrid search weights
    bm25_weight=0.30,
    recency_weight=0.10,
    dedup_threshold=0.85,             # deduplicate above this similarity
    merge_threshold=0.72,             # merge near-duplicates
    decay_rate=0.005,                 # memory decay per day
    async_extraction=True,            # fire-and-forget learn()
    context_window_turns=6,           # conversation turns analysed
    max_context_chars=1500,           # max chars injected into prompt
    debug=False,
)
piq = PersoniqAdapter(llm=llm, config=config)

Works alongside other memory systems

personiq stores data in its own SQLite database and does not interfere with Mem0, Zep, LangMem, or any custom memory setup.

python
piq  = PersoniqAdapter(llm=llm)   # personiq - personalization facts
mem0 = MemoryClient(...)            # Mem0 - conversation history

# both work independently on the same conversation
ctx = piq.persona("alice", message)
hist = mem0.get_history("alice")
💡
personiq focuses on personalization - durable user facts. Tools like Mem0 and Zep handle conversation history. They serve complementary purposes and work together without conflict.
